# AGENTS.md

## Project Shape
- Small Python crawler for Douyin live-web messages; `main.py` is the runnable sample and `liveMan.py` contains the fetcher, WebSocket setup, signing calls, and message parsers.
- Runtime uses both Python and JavaScript: `PyExecJS` executes `a_bogus.js`, and `py_mini_racer` executes `sign.js` for WebSocket signatures.
- `protobuf/douyin.py` is generated by `betterproto`; edit `protobuf/douyin.proto` and regenerate instead of hand-editing generated classes.

## Setup And Run
- Install Python deps with `pip install -r requirements.txt`.
- README-tested environment is Windows 10, Python 3.7+, Node.js v18.2.0, and `protoc` 25.1.
- Run the sample with `python main.py`; change the hard-coded `live_id` in `main.py` or instantiate `DouyinLiveWebFetcher(live_id)` directly.
- There is no `package.json` by design; `.gitignore` excludes `package.json`, `package-lock.json`, and `node_modules/`.

## Verification
- No repo test/lint/typecheck config exists. Use `python -m py_compile main.py liveMan.py ac_signature.py protobuf\douyin.py` as the quickest local syntax check.
- End-to-end verification requires network access to Douyin live endpoints and a currently valid live room ID; failures may be endpoint/signature drift, not just local code bugs.

## Generated And Signing Files
- Regenerate protobuf from inside `protobuf/` with `protoc -I . --python_betterproto_out=. douyin.proto`; `betterproto==2.0.0b6` is pinned and documented as required.
- Current signature path is `generateSignature(..., script_file='sign.js')`, calling JS function `get_sign(md5)` through `MiniRacer`.
- `sign_v0.js` is an older fallback path referenced only in comments; it requires `jsdom` if re-enabled, while the current repo has no Node dependency manifest.
- Avoid formatting or refactoring the obfuscated JS files (`a_bogus.js`, `sign.js`, `sign_v0.js`, `webmssdk.js`) unless the task is specifically about updating the signing logic.

## Code Gotchas
- `DouyinLiveWebFetcher.room_id` scrapes `roomId` from the live page and the code comments note it can fail transiently; retrying may be valid.
- `get_room_status()` is marked invalid in `main.py`, but `_wsOnClose()` still calls it; be careful before relying on it for health checks.
- `_wsOnMessage()` silently ignores parser exceptions around individual messages; adding new message parsing should avoid breaking the whole WebSocket loop.
